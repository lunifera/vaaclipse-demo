Cassandra is the IDE-like demo environment allowing you to browse and learn its own code and demonstrating some Vaaclipse features. You can obtain Cassandra sources from Vaaclipse repository.

Bundles

  • org.semanticsoft.vaaclipsedemo.cassandra.app - application bundle providing product, demo implementation, resources and compiled widgetset.
  • org.semanticsoft.vaaclipsedemo.cassandra.vaadinaddons - provides the vaadin addon binaires used in this demo.

Product configuration

Open file plugin.xml. The extension point org.eclipse.core.runtime.products contains the product configuration. Read about product configuration in Project configuration guid.

  • applicationXMI - this file contains the application model. You can browse it using reach UI in Eclipse Model Editor.
  • applicationWidgetset - the location of application widgetset. As you can see this application uses widgetset is located in widgetset/cassandra.widgetset.CassandraWidgetset.
  • cssTheme - the identifier of theme used by application. Cassandra uses the reindeer theme.
  • applicationHeaderIcon - you can see this icon in the browser window title.

Theme configuration

Open file plugin.xml. The extension point org.semanticsoft.vaaclipse.theme contains the theme configuration. This extension point introduces the theme contribution org.semanticsoft.vaaclipsedemo.cassandra.app.maincontrib that extends the reindeer theme with stylesheet "css/default.css" and resources in "img/". The another contribution org.semanticsoft.vaaclipse.resources contains stylesheet used by Vaaclipse. Read about themes and theme contributions in Resource system guide. Shortly, theme contribution extends the given themes with resources and styles. Theme contributions help us to build modular applications. Contributions are counterparts to theme inheritance (that has more limited use). For example, we can create the new theme with our resources and styles and inherit it from reindeer theme or use the reindeer theme itself extending with our resources and styles. The second way is more preferable (see Contribution vs inheritance section of Resource system guide).

Java code

Package org.semanticsoft.vaaclipsedemo.cassandra.app

  • .views contains the contribution classes for parts. For example, PackageExplorer class contains the contribution for part with elementId = org.semanticsoft.vaaclipsedemo.cassandra.app.part.packageexplorer.
  • .editors contains the contribution classes for editors. The editors in this demo don't allow editing source code, but you can familiarize with editors infrastructure on this sample.
  • .toolitems contains the contributions for Tool Items.
  • .handlers contains the contributions classes for handlers. For example, CollapseAll class contains the contribution for Direct Tool Item with elementId = org.semanticsoft.vaaclipsedemo.cassandra.app.directtoolitem.collapseall.
